<p>ありえるえりあ勉強会@<a class="keyword" href="http://d.hatena.ne.jp/keyword/%CE%AF%C3%D3%BB%B3%B2%A6">溜池山王</a>　〜第1回ディープすぎる<a class="keyword" href="http://d.hatena.ne.jp/keyword/Java">Java</a>・・・〜EffectiveでPerfectな<a class="keyword" href="http://d.hatena.ne.jp/keyword/Java">Java</a>勉強会<br />
<a href="http://atnd.org/events/4261">http://atnd.org/events/4261</a></p><p>そもそも、<a class="keyword" href="http://d.hatena.ne.jp/keyword/IT%CA%D9%B6%AF%B2%F1%A5%AB%A5%EC%A5%F3%A5%C0">IT勉強会カレンダ</a>ーを眺めていて、<a class="keyword" href="http://d.hatena.ne.jp/keyword/Java">Java</a>の勉強会があるっ！！と何の気なしにキャンセル待ちで申し込んだのですが、かなりの参加者がいて、どうせ行けないと思っていました。</p><p>がっ、木曜に確認してみたところキャンセルが続出で、いつのまにか参加者リストに入ってました。そりゃ、行くしかないと思って参戦です笑。</p>

<div class="section">
    <h4>目的</h4>
    
<ul>
<li>東京の勉強会で、懇親会まで参加する</li>
<li><a class="keyword" href="http://d.hatena.ne.jp/keyword/Java">Java</a>勉強中なので、何か知識が増やせたらいいな</li>
<li>Effective <a class="keyword" href="http://d.hatena.ne.jp/keyword/Java">Java</a>の翻訳者の方に会いに行くw</li>
</ul><p>1つ目は、まだ東京の勉強会では、懇親会に参加したことがないのでどんな感じなのかな？っとおそるおそる参加表明。3つ目は、Effective <a class="keyword" href="http://d.hatena.ne.jp/keyword/Java">Java</a>を一度読んだことが会ったってだけですが、どんな人が翻訳してるのかなぁーって思ったのがきっかけです。</p><br />
<p>さっそく、感想。<br />
まずは、Effective <a class="keyword" href="http://d.hatena.ne.jp/keyword/Java">Java</a>の翻訳者、柴田 芳樹さんの講演。</p>

</div>
<div class="section">
    <h4><a class="keyword" href="http://d.hatena.ne.jp/keyword/%A5%D7%A5%ED%A5%B0%A5%E9%A5%DF%A5%F3%A5%B0%B8%C0%B8%EC">プログラミング言語</a><a class="keyword" href="http://d.hatena.ne.jp/keyword/Java">Java</a>教育を振り返って</h4>
    <p>行っている研修の内容、受講者がつまづく点、気をつけるべきことについてのセッションだった。行っている研修とは、</p>

<ul>
<li>教科書は、<a class="keyword" href="http://d.hatena.ne.jp/keyword/%A5%D7%A5%ED%A5%B0%A5%E9%A5%DF%A5%F3%A5%B0%B8%C0%B8%EC">プログラミング言語</a><a class="keyword" href="http://d.hatena.ne.jp/keyword/Java">Java</a>、Effective <a class="keyword" href="http://d.hatena.ne.jp/keyword/Java">Java</a>、Graphic <a class="keyword" href="http://d.hatena.ne.jp/keyword/Java">Java</a></li>
<li>毎回予習で教科書を１００ページ読む</li>
<li>ついてくる練習問題を解き、疑問点を見つける</li>
<li>授業では、疑問点を事前に調査してその疑問点を解決していく（講義形式は行わない）</li>
<li>最終成果物は、デジタル時計</li>
</ul><p>といったものだった。</p><br />
<p>予習は、すべてプライベートな時間に行い、月60〜90時間を費やすとのこと。プライベートの時間を投資するということで、やはりそれだけやらないと身に付かないのだと感じた。<br />
<br />
</p>

<div class="section">
    <h5><a class="keyword" href="http://d.hatena.ne.jp/keyword/GUI">GUI</a>課題としてのデジタル時計</h5>
    <p>AWTだけで作る段階と、Swingだけで作る２段階で構成させている。その２段階それぞれで次のことを行う。</p>

<ol>
<li>シンプルなデジタル時計</li>
<li>ダイアログを開く　フォント、フォントサイズ、背景色を変えられるように(AWTの場合、ちらつきが生じるため、ダブルバッファリングを実装する)</li>
<li>フレーム枠のないデジタル時計を作成　ウィンドウ内をドラックで移動、右クリックでポップアップメニュー</li>
</ol><p><br />
話の中でわからなかった部分を後で調べるために、残しておく。</p>

<ul>
<li>クラスの拡張で「契約」って何？</li>
<li>equalsメソッドとhashCodeメソッドの違い</li>
<li><a class="keyword" href="http://d.hatena.ne.jp/keyword/%C9%E2%C6%B0%BE%AE%BF%F4">浮動小数</a>点周りのstrict</li>
<li>List<Integer>はList<Number>のサブ</li>
<li>マルチスレッド全般</li>
<li>リフレクションって？</li>
<li><a class="keyword" href="http://d.hatena.ne.jp/keyword/GC">GC</a>関連　強い参照、ソフトな参照、弱い参照、ファントム参照って？</li>
</ul><p>ふぅ、ヤバいな全然分かんない>_<</p><br />
<p>次に、アリエル・ネットワークCTOの井上 誠一郎さんの講演。</p>
